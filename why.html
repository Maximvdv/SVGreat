So, why should I be using SVG?

Well, it's SVG, not science...

Introduction

In a world of modern technology, responsiveness is a key feature.
We need to customize everything for a various number of devices so our users can benefit from an optimal user experience every time.
While doing that, we also want to keep our file sizes down for performance.

With the implementation of SVG, you can keep your website pixel-perfect at all resolutions while having small file sizes and still be able to edit and modify your website with its own unique touch.

Properties

  • Supported by all browsers
  • Resolution independant
  • Easy to edit
  • Ideal for custom animations
  • Compatible with CSS and Javascript
  • Highly compressible

So, how do you get an SVG image?

SVG images are formed by either raw code or edited with a specific software. There are a number of different programs that will allow you to create your own graphic designs.

When exported you will be able to choose between a format in raw code or as a .svg file extension. This allows you to use the scalable image in its entire format or edit in the code if you need specific actions to be performed. The more detailed the design, the more classes and lines of code.

Why should you use this instead of other image types?

There's still a lot of discussion why you should SVG over PNG images. They both support images which can have transparancy, so they are ideal for logo or icon usage.

The main reason is that images take up about 60% of your website's memory. So you want this to perform well, but without losing too much of your site's speed. SVG offers the solution for your problem, because it can be minified to a few Kb's of data, without losing any quality when scaling up because it's vector based.

What do you mean with "dynamic SVG"?

SVG allows you to access the data directly from the DOM. This way you can get real-time edits combined with input via CSS and Javascript.

The advantage that this offers in contrary to its counterparts, is that you can make the data as big and scalable as you want, without having your scripts taking up a lot of memory.
